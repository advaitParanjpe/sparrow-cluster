# Sparrow-Cluster

Milestone 8 adds a frozen SparrowML WISDM deployment package as the primary end-to-end multicore application workload on top of the four production coherent L1D caches and minimal RV32 `LR.W`/`SC.W`. The system remains four Sparrow-V cores with private L1I, private coherent write-back/write-allocate L1D, one serialized snoopy transport, and one shared SRAM. Atomics are still limited to one 16-byte-block reservation per core and do not include other RV32A AMOs.

The normal simulation is self-contained. `scripts/import_sparrow_v.sh ../sparrow-v` is only the explicit provenance refresh operation; it copies the reviewed scalar closure and is not required to run tests. Milestone 7 software images are generated by `scripts/build_runtime_sw.py` into ignored `build/sw/` because the local environment does not provide a complete RISC-V ELF toolchain.

# Boot and runtime

All four cores start together at zero. Milestone 7 images jump from reset to generated runtime text at `0x800`; each hart reads uncached `0x10000000`, calculates a distinct cacheable stack below `0x10000`, and records its participation. Hart 0 initializes shared runtime data and publishes the uncached release flag at `0x308`; secondary harts wait for that release before entering the selected workload.

Results and completion words use uncached apertures for deterministic observation. Shared workload data starts at `0x1000` and uses ordinary cacheable coherent accesses. Inactive harts publish safe completion without modifying active workload results.

The executable flow is generated by `scripts/build_runtime_sw.py` rather than a C compiler. `sw/runtime/` and `sw/linker/sparrow_cluster.ld` define the intended libc-free startup, linker symbols, and runtime ABI for a later ELF flow.

Milestone 8 SparrowML images use the same boot, active-hart filtering, uncached release/completion words, and cacheable coherent data path. The SparrowML region at `0x3000` is above generated text and below the 64 KiB testbench stack area, with safe-layout output blocks and shared-work partial blocks aligned to 16-byte cache blocks.

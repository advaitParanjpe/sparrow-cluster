# Verification plan

`make sim-unit` preserves reset, continuous simultaneous four-way round-robin rotation, controller fixed read/write latency, source retention, byte enables, invalid access response, and adapter IMEM capture/response routing. `make sim-l1i` covers cold four-word refill, all word offsets, hit after refill, same-set two-way occupancy, deterministic third-block replacement, delayed lower responses, reset invalidation, and counter consistency. RTL assertions cover valid-way hits, aligned/in-range refills, and tracked responses.

`make sim-cluster` runs simultaneous four-core cold L1I misses through shared fetch, hart-ID load, and result stores, and checks each L1I counter relationship. `make sim-multicore` image-loads a bare-metal RV32I program through L1I and validates all four IDs, stack locations, release synchronization, checksums, word/byte partition writes, completion words, L1I hits, and refill accounting. `make regress` runs all checks and simulations. The test suite uses Icarus Verilog; its informational constant-select warnings are not simulation failures.

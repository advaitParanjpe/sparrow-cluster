# Cache architecture

Milestone 2 implements L1I only. Each L1I is 2 KiB, 2-way, 64 sets, and 16-byte blocks: `64*2*16=2048`. With the audited 32-bit address interface, byte offset is `[1:0]`, word offset is `[3:2]`, set is `[9:4]`, and tag is `[31:10]` (22 bits). One replacement bit per set selects the victim. Reset clears valid and replacement bits; an invalid way is chosen first (way 0 before way 1), and a full set uses the bit. A hit or fill sets the bit to the other way, a one-bit pseudo-LRU/round-robin convention.

L1I is blocking, read-only, private, and non-coherent: one fetch or refill is active, with no MSHR or hit-under-miss. A miss captures the address and victim, issues four aligned 32-bit reads in offset order, installs tag/valid only after word four, then returns the requested word. Data and tags need not be reset because cleared valid bits suppress them. There is no flush, invalidate, self-modifying-code support, or instruction/data coherence; program memory is immutable after execution begins.

The processor side is the audited 32-bit IMEM valid/ready contract. The L1I lower side is the existing 32-bit adapter IMEM contract, not a new bus: each read carries one aligned word address and receives one word response. Per-core internal counters expose accesses, hits, misses, refill words, and miss-stall cycles; accesses are counted on accepted fetches, so `accesses = hits + misses`.
